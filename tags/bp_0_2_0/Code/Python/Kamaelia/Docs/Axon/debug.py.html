Axon.debug.py

Version: Axon 1.0

Simple debugging class. When dealing with single threaded concurrency it can become difficult to trace code, limiting the effectiveness of debuggers. The purpose of this debug subsystem is to allow a configurable method for debugging. Prior to changing to a setup.py based method of usage, this worked very well (it would read a debug config file from the current directory), but in the current incarnation this has limited use. For now if the config file can't be found, debugging is disabled.

There are a number of issues with the way debugging is handled at present:

  • Configuration and defaults
  • The extremely heavy hit that switching on debug causes on the system. (Generally you want to run python with -OO flags as a result to get any sort of speed)

Essentially the debugging divides the system into sections, and each section can have a debug level. The higher the debug level, the more output for that section you see. Debugging can also be switched on on a per component basis.

debug(self, section, level, *message)

  • Outputs *message if user set level is greater than requested level for given section returns True. This allows debug to be used in assert statements to allow lazy evaluation of expressions in *message so that they can disabled by running the system using python's -O flag

note = debug(self, section, level, *message)

  • note is an alias for debug.
