This document is a work in practice!

Setting up the development environment.

This is recognised as being currently too complicated, but this document serves as to document how to get started initially.

In the following I'm assuming you're using any modern generic

linux, and have the ability to run programs as root (either by

being root, or via sudo).

Installing from tar balls (ie last release ~ October)

=====================================================

You need the following files for a minimal install:

Axon-1.1.2.tar.gz

Kamaelia-0.3.0.tar.gz

Once you have these:

tar zxvf Axon-1.1.2.tar.gz

cd Axon-1.1.2

# If you're logged in as root, do this:

python setup.py install

# If you have sudo access instead, do this:

sudo python setup.py install

# Change back to where the tar balls are:

cd ..

tar zxvf Kamaelia-0.3.0.tar.gz

cd Kamaelia-0.3.0

# If you're logged in as root, do this:

python setup.py install

# If you have sudo access instead, do this:

sudo python setup.py install

This will then allow you to play with the system, however a

number of the examples, and demos use a variety of media and

libraries. The following will describe what you need for

each of the more interesting/useful dependencies.

Installing from CVS (preferred)

===============================

mkdir Kamaelia

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/kamaelia login

# You'll be prompted for a password with the above, and since it doesn't

# have one, press return.

cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/kamaelia co -P .

cd Kamaelia

KAMAELIAROOT=`pwd`

cd Code/Python/Axon

sudo python setup.py install

cd $KAMAELIAROOT

cd Code/Python/Kamaelia

sudo python setup.py install

Installing optional dependencies

================================

PYGAME

------

If you're using a modern install of Linux, there's a high

chance that pygame is installed. This provides some relatively

low level/simple tools for driving the display - providing

tools like sprites, canvases, etc - and is used by a number

of the tools we use.

If it's not installed it's worth checking to see if it's included

in your distribution. I know it's available in SuSE, Redhat/Fedora,

Mandrake, and a bunch of others.

In case it isn't you can download it from here:

http://www.pygame.org/download.shtml

Installation manually is done as follows:

tar zxvf pygame-1.7.1release.tar.gz

cd pygame-1.7.1release

# if you have root access:

python setup.py install

# otherwise if you have sudo access:

sudo python setup.py install

PYAO

----

If you want to use the vorbis examples, then you need something to

output the audio. PyAO wraps libao which is an audio output library.

This is generally included in most linux distributions and if it's

not installed can be quickly/simply installed.

PYAO itself though is not commonly included, and as a result needs

to be grabbed and installed if you want to use the components that

wrap pyao.

This can be downloaded from here:

http://www.andrewchatham.com/pyogg/

Specifically:

http://www.andrewchatham.com/pyogg/download/pyao-0.82.tar.gz

Install as follows:

tar zxvf pyao-0.82.tar.gz

cd pyao-0.82

python config_unix.py

# then if you have root access:

python setup.py install

# otherwise if you have sudo access:

sudo python setup.py install

PYREX

-----

For many of the libraries we wrap, we use a tool called Pyrex. This

provides a python-like language that interfaces directly with C whilst

simplifying making C functionality available at a high level. (I also

think it's a really cool little tool/language for what it does, but

that's a different matter :-)

Pyrex's homepage is here:

http://www.cosc.canterbury.ac.nz/~greg/python/Pyrex/

The version we use is the latest one, which can be downloaded here:

http://www.cosc.canterbury.ac.nz/~greg/python/Pyrex/Pyrex-0.9.3.1.tar.gz

Installation:

tar zxvf Pyrex-0.9.3.1.tar.gz

cd Pyrex-0.9.3.1

# The if you have root access:

python setup.py install

# otherwise if you have sudo access:

sudo python setup.py install

VORBISSIMPLE

------------

Similarly in order to use the vorbis examples, you need to install

libogg and libvorbis. I don't know a linux distribution that doesn't

include these. In order to wrap vorbis, we needed to build a simple

C wrapper library, and some python bindings for this.

If you're installing from CVS, you can do this to install the simple

C wrapper library:

cd $KAMAELIAROOT

cd Sketches/Vorbis/vorbissimple/libvorbissimple/

./configure

make

# as root:

make install

ldconfig

# or if you only have sudo:

sudo make install

sudo /sbin/ldconfig

Then to install the python bindings

cd ..

cd python

# then as root, if you have root access:

python setup.py install

# Otherwise if you only sudo access instead:

sudo python setup.py install

If you're installing from tarballs, rather than CVS, then you need the

following tar balls:

vorbissimple-0.0.1.tar.gz

Which you install as follows:

tar zxvf vorbissimple-0.0.1.tar.gz

cd vorbissimple-0.0.1

# There's installation information in this directory, but the

# quick version is:

cd libvorbissimple

./configure

make

make install (or sudo make install if you have sudo access)

/sbin/ldconfig (or sudo /sbin/ldconfig if you have sudo access)

DIRAC

-----

If you're interested in playing with the Dirac components for video

playback (or even encoding :-), then you need to install Dirac, and

our bindings for dirac.

For this you need to install dirac first. This can be got from the

Dirac downloads, and installed in the same way as others. From

http://sourceforge.net/project/showfiles.php?group_id=102564

grab:

dirac-0.5.4.tar.gz

To install:

tar zxvf dirac-0.5.4.tar.gz

cd dirac-0.5.4

./configure

make

# **Takes a while**

# Then as root, if you have root access:

make install

ldconfig

# if you have sudo access instead of root:

sudo make install

sudo /sbin/ldconfig

To install the python bindings for dirac that we use for the

Dirac components...

Installation from CVS is again simplest in my opinion:

cd $KAMAELIAROOT

cd Code/Python/Bindings/Dirac

# As root:

python setup.py install

# or for sudo

sudo python setup.py install

Installation from the tar ball - grab from the kamaelia downloads page:

Dirac-0.0.1.tar.gz

tar zxvf Dirac-0.0.1.tar.gz

cd Dirac-0.0.1

# Then as root:

python setup.py install

# Or if you have sudo access instead:

sudo python setup.py install

DVB (Freeview dumping)

Finally if you really want to play with the DVB components forming

in /Sketches/MPS/DVB/Components, then you might want the python-dvb

bindings which are in the download:

python-dvb3-0.0.4.tar.gz

Installation is pretty much the same as the rest:

tar zxvf python-dvb3-0.0.4.tar.gz

cd python-dvb3-0.0.4u

# Then as root

python setup.py install

# Or using sudo

sudo python setup.py install

If you're using Linux and want to play with these, the the freecom DVB-T

USB stick works very nicely.
